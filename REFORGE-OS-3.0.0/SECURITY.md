# Security Policy

## Scope

REFORGE OS is a compliance-first analysis platform for lawful recovery only.
It does not execute circumvention.

---

## Reporting Security Issues

Security issues should be reported privately to security@reforge-os.com

**Do not open public issues for security vulnerabilities.**

---

## Data Handling

✅ **Collected**:
- Account identity and role
- Device metadata (non-content)
- Ownership verification artifacts (hashes/flags)
- Audit logs (action labels, timestamps)

❌ **NOT Collected**:
- Device contents
- Credentials
- Exploit code
- Binaries or scripts

---

## Internal Separation

**Pandora Codex** exists as internal-only knowledge.
Any attempt to expose it publicly is a security violation.

---

## Access Controls

- Role-based access
- Hardware-serial binding
- Multi-factor authentication
- Immutable audit logs

---

## Compliance

All security measures comply with:
- GDPR
- CCPA
- Regional data protection laws
- Industry security standards

---

## Responsible Disclosure

We follow responsible disclosure practices.
Security researchers who report vulnerabilities will be credited appropriately.

---

**Security is a priority, not an afterthought.**
