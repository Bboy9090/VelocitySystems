## THE SYSTEM IN ONE PAGE

### Core Flow

```
INTAKE → CONSENT → SGDE DIAGNOSTICS → TRUST GATE
     → STRESS TESTS → DECISION (SAFE/RISK/STOP)
     → REPAIR (IF APPROVED) → POST-VALIDATION
     → PHOENIX FORGE CERTIFICATE → LEDGER CLOSE
```

### Key Principles

1. **Gate Enforcement** — No step without evidence
2. **Evidence-First** — No guessing, no opinions
3. **Risk Disclosure** — Before action, not after
4. **Immutable Ledger** — Append-only, hash-verified
5. **Stop is Valid** — Refusal is professionalism

### Platform Engines

- **SGDE-A (Apple)** — macOS trust, iOS assessment, calibration
- **SGDE-Android** — Bootloader, OEM security, account locks
- **SGDE-W (Windows)** — Firmware, TPM, driver coherence
- **SGDE-C (Consoles)** — Ban-risk, pairing, thermal stability

### Outputs

- **Trust State** — Platform-specific trust score (0-100)
- **Risk Assessment** — Lockout, feature loss, data loss, irreversibility
- **Readiness Index** — Composite health score (0-100)
- **Phoenix Forge Certificate** — Immutable, hash-verified
